Closed the honesty layer's remaining silent-drop blind spots and added property-based fuzzing to prove it stays closed. Message create, webhook execute, interaction responses and bulk message delete now route their bodies through `ctx.fields` like every edit, so an unmodelled key (e.g. `sticker_ids`) fails loudly with `UnsupportedField` instead of vanishing. A Hypothesis sweep proves that across every route whose body flows through `ctx.fields`/`list_fields`, an unrecognised request field always raises and a recognised one never does, and a drift guard fails loudly if a new write route is neither honesty-vetted nor explicitly exempted with a reason.
